# Agent Cards

These files are the runnable agent cards for the KIO Site Finder implementation model.

## Startup Order

1. `orchestrator_agent.md`
2. `requirements_agent.md`
3. `architecture_agent.md`
4. Domain owner card for the current sprint
5. `qa_agent.md`
6. `ops_release_agent.md` when deployment or environment work is involved

## Required Shared References

- `../agent_execution_playbook.md`
- `../delivery_controls_pack.md`
- `../master_delivery_plan.md`
- `../SRS.md`
- `../SAD.md`
- `../SDD.md`

## Rule

No agent card should be used without a sprint ID, an owning artifact list, and a named downstream handoff target.
